# Risk Profile Output XML Schema Documentation

## Overview

This XML Schema Definition (XSD) validates the output XML files generated by the ILF Evaluation Server. The schema defines the structure, data types, and constraints for risk profile evaluation results.

## Schema File

**Location**: `schema/risk_profile_output.xsd`

## Validation

### Validate XML Against Schema

```bash
# Basic validation
xmllint --noout --schema schema/risk_profile_output.xsd server_output.xml

# With detailed output
xmllint --schema schema/risk_profile_output.xsd server_output.xml

# Validate and format
xmllint --format --schema schema/risk_profile_output.xsd server_output.xml
```

### Expected Output

```
server_output.xml validates
```

## Schema Structure

### Root Element

```xml
<RiskProfiles>
  <RiskProfile>...</RiskProfile>
  <RiskProfile>...</RiskProfile>
  ...
</RiskProfiles>
```

### RiskProfile Element

The `RiskProfile` element contains all information about a single risk profile, including:

#### Basic Information
| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `risiko_name` | string | No | Risk category name (e.g., "Risiko Kredit") |
| `Profile_ID` | ProfileIDType | No | Unique identifier (e.g., "RI0100100000") |
| `Faktor_Penilaian` | string | No | Assessment factor description |
| `Derived_Source` | string | No | Data source ("Risk Profile" or "data source") |

#### Calculation Logic
| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `Threshold` | string | No | Threshold ranges for rating determination |
| `ThresholdFunction` | string (CDATA) | No | C code for threshold calculation |
| `Logic_Risk_Profile` | string | No | Description of calculation logic |
| `Logic_Risk_Profile_Fx` | string (CDATA) | No | C code for risk profile calculation |
| `Input_System` | string | No | Input system identifier |
| `Sandi_Acuan` | string | No | Reference code |

#### Weighting
| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `Sample_Bobot` | decimal or empty | No | Weight for weighted calculations (e.g., 0.28) |
| `Sample_Hasil_Perhitungan` | string | No | Sample calculation result |
| `Sample_Score` | string | No | Sample score value |
| `Bobot` | decimal or empty | No | Alternative weight field |

#### Evaluation Results
| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `riskProfileEvaluationResult` | EvaluationResultType | No | Score from Logic_Risk_Profile_Fx |
| `thresholdRating` | RatingType | No | Rating (1-5) from ThresholdFunction |
| `finalScore` | ScoreType | No | Score mapped from rating |
| `scoreXweight` | WeightedScoreType | No | finalScore × Sample_Bobot |

#### Additional Metadata
| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `Kriteria` | string | No | Criteria description |
| `Parameter` | string | No | Parameter information |

## Complex Types

### EvaluationResultType

```xml
<riskProfileEvaluationResult>
  <value>36.200000</value>
</riskProfileEvaluationResult>
```

Contains a single `value` element with a double precision number representing the evaluation score.

### RatingType

```xml
<thresholdRating>
  <value>4.0</value>
</thresholdRating>
```

Contains a `value` element with a rating between 1.0 and 5.0, where:
- **5.0**: Lowest risk (Score ≤ 36)
- **4.0**: Low risk (36 < Score ≤ 52)
- **3.0**: Medium risk (52 < Score ≤ 68)
- **2.0**: High risk (68 < Score ≤ 84)
- **1.0**: Highest risk (84 < Score ≤ 100)

### ScoreType

```xml
<finalScore>
  <value>44.00</value>
</finalScore>
```

Contains a `value` element with the final score mapped from the rating using `rating_to_score.xml`.

### WeightedScoreType

```xml
<scoreXweight>
  <value>12.320000</value>
</scoreXweight>
```

Contains a `value` element with the weighted score calculated as: `finalScore × Sample_Bobot`

## Simple Types

### ProfileIDType

**Pattern**: `(RI[0-9]{10})?`

- Must start with "RI" followed by exactly 10 digits
- Example: `RI0100100000`
- Can be empty
- Hierarchical: parent profiles end with "0000"

**Valid Examples**:
- `RI0100100000` (parent)
- `RI0100100100` (child)
- `RI0200300000` (another parent)
- `` (empty - valid)

**Invalid Examples**:
- `RI01001` (too short)
- `RI010010000000` (too long)
- `AB0100100000` (wrong prefix)
- `RI01001ABCDE` (non-numeric)

### RatingValueType

**Constraint**: Double between 1.0 and 5.0 (inclusive)

Represents the risk rating on a 5-point scale with inverse relationship to risk:
- Higher rating = Lower risk
- Lower rating = Higher risk

### OptionalDecimalType

Union type that allows either:
- A valid decimal number
- An empty string

Used for fields like `Sample_Bobot` and `Bobot` which may not always have values.

## Schema Features

### Flexible Element Ordering

The schema uses `xs:all` instead of `xs:sequence`, allowing elements to appear in any order within a `RiskProfile`. This provides flexibility for different data sources and evaluation scenarios.

### Optional Elements

All child elements of `RiskProfile` are optional (`minOccurs="0"`), allowing for:
- Partial risk profiles
- Profiles at different evaluation stages
- Profiles with varying data availability

### CDATA Support

Elements like `ThresholdFunction` and `Logic_Risk_Profile_Fx` are defined as strings to support CDATA sections containing C code.

### Empty Values

Several fields support empty values through:
- `OptionalDecimalType` for numeric fields
- Pattern `(pattern)?` for Profile_ID
- Optional elements throughout

## Validation Errors

### Common Validation Errors

#### 1. Invalid Profile_ID Pattern
```
Element 'Profile_ID': [facet 'pattern'] The value 'RI01001' is not accepted by the pattern 'RI[0-9]{10}'
```
**Solution**: Ensure Profile_ID has exactly 12 characters (RI + 10 digits)

#### 2. Invalid Rating Value
```
Element 'value': '6.0' is not a valid value of the local atomic type
```
**Solution**: Rating must be between 1.0 and 5.0

#### 3. Invalid Decimal
```
Element 'Sample_Bobot': 'abc' is not a valid value of the atomic type 'xs:decimal'
```
**Solution**: Use numeric values or leave empty

## Integration with ILF Server

### Generated by Server

The XML validated by this schema is automatically generated by:
1. `eval_cli` - CLI tool
2. `eval_server` - Unix socket server

### Pipeline Output

The schema validates the final output after complete pipeline execution:
```
Input XML → Evaluation → Threshold → Rating Mapping → Weighted Calculation → Output XML (validated by schema)
```

### Usage in Applications

```python
# Python example with lxml
from lxml import etree

# Parse schema
schema_doc = etree.parse('schema/risk_profile_output.xsd')
schema = etree.XMLSchema(schema_doc)

# Validate XML
xml_doc = etree.parse('server_output.xml')
is_valid = schema.validate(xml_doc)

if not is_valid:
    print(schema.error_log)
```

## Best Practices

### 1. Always Validate Output

```bash
# In scripts
if xmllint --noout --schema schema/risk_profile_output.xsd output.xml 2>&1 | grep -q "validates"; then
    echo "✓ Valid XML"
else
    echo "✗ Invalid XML"
    exit 1
fi
```
